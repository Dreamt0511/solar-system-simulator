# 太阳系模拟器 — 视觉增强设计

## 概述

在现有太阳系模拟器基础上增加两项视觉增强：程序化银河系背景和地球专属的体积云+高级大气散射。两项均与现有功能共存，不破坏已有行为。

---

## 一、程序化银河系背景

### 目标

用粒子系统生成三维银河系旋臂作为场景背景，替代/补充当前的静态星空贴图，从任何视角观察都能看到银河盘面结构。

### 实现方案

新文件 `js/milkyWayGalaxy.js`，导出 `MilkyWayGalaxy` 类。

#### 粒子系统

- 使用 `THREE.Points`，约 15000-20000 个粒子
- 粒子分布为**双旋臂螺旋结构**：
  - 按对数螺线方程生成坐标：`r = a * exp(b * θ)`，两个旋臂相位差 180°
  - 盘面厚度从中心向边缘渐变（中心厚，边缘薄），给银河一个"透镜"状截面
  - 在旋臂周围散布一些随机晕粒子（弥散晕）
- 粒子颜色渐变：
  - 核心（黄色/橙黄，`0xffdd66`）
  - 旋臂内部（暖白/淡黄）
  - 旋臂外部和晕（蓝白，`0x8888ff`）
- 粒子大小：核心较大，向外递减

#### 放置与视角

- 银河放置在场景原点，盘面在 XZ 平面（`rotation.x = Math.PI / 2`）
- 工作半径约 800-1200 单位（远大于太阳系范围），始终包围玩家视角
- 银河整体绕 Y 轴缓慢自转（周期约 120 秒真实时间）

#### 与现有 "美丽星空" 模式的共存

两者**互斥切换**，避免视觉冲突：

- **默认状态**：程序化银河可见，银河替代 `scene.background`（设为纯黑）
- **用户点击"美丽星空"**：隐藏银河粒子系统，切换 `scene.background` 为深空图片
- **用户关闭"美丽星空"**：显示银河粒子系统，`scene.background` 恢复为纯黑
- 具体实现：在 `MilkyWayGalaxy` 上提供 `visible` 属性，`StarfieldBackground.show()` 时设置 `galaxy.visible = false`，`hide()` 时恢复 `true`

#### 接口

```js
class MilkyWayGalaxy {
  constructor(scene) { /* 创建粒子系统并添加到 scene */ }
  update(time) { /* 缓慢自转 */ }
  dispose() { /* 释放几何和材质 */ }
}
```

### 影响范围

| 文件 | 改动 |
|------|------|
| `js/milkyWayGalaxy.js` | 新增 |
| `js/main.js` | 初始化时创建 `MilkyWayGalaxy` 实例，动画循环中调用 `update()` |

## 二、地球体积云与大气散射

### 目标

为地球增加两层视觉增强：半透明旋转云层 + 基于物理的瑞利大气散射，使地球从太空视角更真实。

### 实现方案

在现有 `js/planetDetails.js` 中扩展，新增云层系统和升级大气散射着色器。

#### 2.1 云层系统

- 在地球外围增加一个稍大的半透明球体（`radius * 1.02`），使用云纹理
- 云纹理来源：生成一张程序化云纹理（Canvas），或使用现有纹理图集中的可用空间
- 云层材质：`MeshBasicMaterial`（自发光，不依赖光照），透明
- 云层绕 Y 轴以稍快于地球自转的速度旋转（模拟大气流动），旋转方向与地球一致
- 纯色模式下云层保持显示（地球颜色 + 白色云纹）
- 纹理模式下云层叠加在地球纹理之上

#### 2.2 高级大气散射

为地球创建新的 Rayleigh 散射大气光晕（当前 `createAtmosphere` 未在任何地方被调用，大气效果为零）。

核心效果：
1. 从太空看地球，边缘有一圈**蓝色光晕**（瑞利散射）
2. 在晨昏线（明暗交界）附近，光晕偏向**橙红色**（米氏散射 / 更长光路）
3. 散射强度随视角和太阳方向动态变化

着色器设计：

```
输入: 视角方向, 太阳方向, 法线
输出: 
  - 大气边缘发光颜色（蓝→橙渐变）
  - 透明度随 Fresnel 效应增强

核心公式: 
  - 散射强度 ∝ 1 / λ⁴（蓝光散射更强）→ 蓝色日侧
  - 光路长度计算 → 视角穿过大气的路径越长越红（日落效果）
  - 结合太阳夹角和视角夹角共同决定最终颜色
```

#### 2.3 结构变更

`planetDetails.js` 现有结构：

```
createPlanetDetails(scene, planets)
  → earthNightShader（已有）
  → 新增: earthCloudMesh
  → 新增: earthAtmosphere

updatePlanetDetails(details, time)
  → 更新太阳方向（已有）
  → 新增: 云层旋转
  → 新增: 大气散射更新
```

#### 2.4 纹理需求

- 使用程序化生成：用 Canvas 生成 Perlin 风格的灰度云纹理，白色=云，黑色=透明，无需额外纹理文件

### 影响范围

| 文件 | 改动 |
|------|------|
| `js/planetDetails.js` | 增加云层 Mesh 创建、大气散射 Mesh 创建；`updatePlanetDetails` 中增加更新逻辑 |
| `js/shaders.js` | 新增地球大气散射着色器（earthAtmosphereVS/FS） |
| `js/main.js` | 无改动（已有 `createPlanetDetails` 和 `updatePlanetDetails` 调用） |
| `js/materialSwitcher.js` | 云层在纹理/纯色模式下均应可见，可能需调整确保兼容 |
| `textures/` | 可选：新增 `earth_clouds.jpg` 云纹理 |

## 三、非功能性要求

- 所有新增效果不显著影响帧率（移动端 30fps+ 可运行）
- 云层粒子数控制在合理范围
- 不使用外部 CDN，纹理/资源本地化

## 四、实现顺序

1. **程序化银河背景**（独立模块，无依赖）
2. **地球云层**（独立，在 planetDetails 中新增）
3. **大气散射升级**（需要确保与云层视觉协调）
